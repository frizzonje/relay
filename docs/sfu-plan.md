# Phase 2 — переезд на SFU (видео на 7 человек «как в Discord»)

> Статус: **план, не реализовано.** Phase 1 (укрепление текущего mesh) сделана —
> см. `apps/web/lib/voice.ts`. Этот документ — про следующий шаг: честное
> видео+экран+голос на 7 участников.

## Почему mesh не вывозит 7 с видео

Сейчас связь **mesh**: каждый шлёт свой поток каждому напрямую. На N участниках
каждый клиент держит N−1 соединений и аплоадит своё видео N−1 раз.

| N | исходящих видео | аплинк @2.5 Мбит/с | декодеров |
|---|----------------|--------------------|-----------|
| 2 | 1              | 2.5 Мбит/с         | 1         |
| 4 | 3              | 7.5 Мбит/с         | 3         |
| 7 | 6              | **15 Мбит/с**      | 6         |

15 Мбит/с вверх стабильно не тянет ни домашний, ни тем более мобильный аплинк.
Голос на 7 mesh держит (6×~64–128 кбит/с), а видео — нет. Это потолок самой
схемы, кодом не лечится.

## Решение: SFU

**SFU** (Selective Forwarding Unit) — медиасервер: каждый шлёт свой поток на
сервер **один раз**, сервер раздаёт остальным. Аплинк участника = константа
(1 видео + 1 звук), сколько бы ни было людей. Так устроены Discord, Meet, Zoom.

Бонусы, которые mesh не даёт в принципе:
- **simulcast** — клиент шлёт видео в 2–3 качествах, сервер раздаёт каждому
  подходящее (мелкая плитка → низкое, на весь экран → высокое);
- серверная запись звонков, при желании;
- единая точка для TURN/ICE — меньше дыр в NAT.

## Выбор: LiveKit (рекомендую)

| | LiveKit | mediasoup | ion-sfu |
|--|---------|-----------|---------|
| Уровень | готовый сервер + SDK | библиотека (писать сигналинг самим) | сервер |
| Self-host | да, один Go-бинарь/контейнер | да | да |
| Клиент | `livekit-client` (React-хуки) | руками | руками |
| Simulcast | из коробки | руками | да |
| Усилий на переезд | **низко** | высоко | средне |

**Рекомендация — LiveKit, self-hosted на вашем сервачке.** Один контейнер,
открытый исходник, держит наши 7 человек с огромным запасом, есть готовый
React-клиент. mediasoup мощнее и гибче, но это «собери SFU сам» — лишние недели.

## Что меняется, что остаётся

**Остаётся (фронт):** гейт пароля, чат/реакции, UI плиток/контролов, микшер
громкости 0–200% (LiveKit отдаёт `MediaStreamTrack`, наш Web Audio-микшер из
Phase 1 ложится поверх как есть).

**Меняется (фронт):** `apps/web/lib/voice.ts` — вместо ручного
`RTCPeerConnection`-mesh и perfect-negotiation используем `livekit-client`:
`room.connect(url, token)`, `room.localParticipant.enableCameraAndMicrophone()`,
подписка на `RoomEvent.TrackSubscribed` → отдаём трек в нашу плитку/микшер.
Сигналинг offer/answer/ICE уходит целиком — его берёт на себя LiveKit.

**Меняется (бэк):** `signaling.gateway.ts` перестаёт ретранслировать WebRTC —
вместо этого NestJS выдаёт **access-token** (JWT с правами на комнату). Чат,
реакции, presence остаются на нашем socket.io как сейчас.

**Инфра:** добавляем сервис `livekit` в `docker-compose.yml`; Caddy проксирует
его WS/HTTP; coturn переиспользуем (LiveKit умеет внешний TURN). Порты UDP для
медиа открыть на сервере.

## Шаги

1. Поднять LiveKit-контейнер локально (`livekit-server --dev`), убедиться, что
   2 вкладки видят друг друга.
2. Бэк: эндпоинт `GET /api/rtc-token?room=...` — выдаёт JWT (API key/secret
   LiveKit в env, рядом с `TURN_CREDENTIAL`).
3. Фронт: переписать `voice.ts` на `livekit-client` за тем же публичным API
   (`joinVoice/leaveVoice/toggleMic/toggleCamera/toggleScreen`), чтобы UI не
   трогать. Микшер громкости и плитки — поверх треков LiveKit.
4. Simulcast + адаптивная подписка (качество по размеру плитки).
5. Прод: сервис в compose, Caddy-маршрут, проверка на 7 реальных людях.
6. Снести mesh-код из `voice.ts` после переезда.

## Чего это стоит

Сервачку 7 человек — копейки (LiveKit для такого масштаба ест мало CPU/RAM).
Главные расходы — аплинк сервера: ~7 входящих + раздача, но при simulcast и
адекватном битрейте укладывается в обычный VPS-канал. Усилий: ~переезд фронт-
слоя связи + токен-эндпоинт + инфра; UI не трогается.

## Открытый вопрос к владельцу

- Self-host LiveKit на вашем сервере (бесплатно, наш контроль) **или** LiveKit

  self-host.
